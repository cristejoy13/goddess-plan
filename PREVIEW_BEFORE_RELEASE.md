# Preview Before Release Rule

Use this as a standing instruction in MiniMax or any other AI coding assistant:

Before any commit, push, PR, deployment, publication, or release action, provide a preview I can open and test.

For web apps, give me a working local preview/review link, such as http://127.0.0.1:<port>/, and tell me exactly what to try. If the project cannot be previewed in a browser, provide the closest usable review artifact, such as screenshots, rendered documents, logs, CLI output, or a staging URL, and explain the limitation.

Run the relevant validation checks before asking for approval. Do not treat a build-only check as my approval.

Wait for my explicit approval before committing, pushing, opening a PR, deploying, publishing, or releasing anything.

Use this wording when ready: Preview is ready: <url>. Please try <specific workflow>. I will wait for your approval before committing or deploying.
